# 账号检测模块接口文档

面向 S2A Manager「账号检测」页：同步范围、用量/鉴权检测、删除与状态开关。  
前端（Tk GUI）直接调用 sub2api 管理端；下列 JSON 为工具调用管理端时的请求/解析约定，以及操作完成后写回界面结果区的汇总结构。

## 认证

请求头（二选一主路径）：

- `x-api-key: <管理员 API Key>`（GUI 默认）
- 或 `Authorization: Bearer <token>`

## 1. 列出账号（同步缓存）

- **GET** `/admin/accounts?page={page}&page_size={page_size}&...`
- **响应 data 要点**：

```json
{
  "items": [
    {
      "id": 101,
      "name": "acc-1",
      "platform": "openai",
      "type": "oauth",
      "status": "active",
      "group_ids": [1, 2]
    }
  ],
  "total": 100
}
```

当前检测分组：对 `items` 在本地按 `group_ids` 过滤；分组为「不限」时不过滤。

## 2. 账号用量（额度检测）

- **GET** `/admin/accounts/{id}/usage`
- **GET** `/admin/accounts/{id}/usage?source=passive|active`（Anthropic oauth/setup-token 可选）

判定达阈值时读取窗口利用率等字段（如 `five_hour` / `seven_day` 等），与界面「5小时阈值(%)」「7天阈值(%)」比较。

## 3. 账号探测（401/403 等）

- **POST** 或 **GET** `/admin/accounts/{id}/test`（以现网 sub2api 实现为准；工具侧按现有 test 调用）

## 4. 批量更新账号状态（关闭 / 开启 / 回写 error）

- **POST** `/admin/accounts/bulk-update`

### 请求

```json
{
  "account_ids": [101, 102],
  "status": "inactive",
  "schedulable": false
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| account_ids | int[] | 目标账号 ID |
| status | string | `active` 开启 / `inactive` 关闭 / `error` 异常 |
| schedulable | bool | 是否参与调度；关闭时建议 `false`，开启时建议 `true`（否则仅改 status 可能看起来仍可调度） |

### 工具期望的成功响应 data（宽松兼容）

```json
{
  "success_ids": [101, 102],
  "results": [
    {"account_id": 101, "success": true},
    {"account_id": 102, "success": true}
  ]
}
```

### 界面结果区汇总 JSON 示例

```json
{
  "updated_ids": [101, 102],
  "failed": [],
  "updated_count": 2,
  "failed_count": 0,
  "status": "inactive",
  "requested_count": 2,
  "scope": "当前检测分组：[1] default (openai/active/公开)",
  "synced_account_count": 50
}
```

### GUI 操作与 status 映射

| 按钮 | status | 账号范围 |
|------|--------|----------|
| 关闭达阈值账号 | inactive | 最近一次无额度检测结果缓存 |
| 全部关闭 | inactive | 当前检测分组内全部账号 |
| 全部开启 | active | 当前检测分组内全部账号 |
| （内部）鉴权异常回写 | error | 检测流程命中的鉴权异常账号 |

## 5. 删除账号

- **DELETE** `/admin/accounts/{id}`

GUI「直接删除* / 删除当前列表」对每个 ID 调用；支持删除并发配置。

## 6. 导出/导入相关

账号数据打包见导出/导入页，路径示例：

- **GET** `/admin/accounts/data?...`
- **POST** `/admin/accounts/data`
